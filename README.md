## 🛠️ Topics Covered

📌 Module 1 — Fundamentals
- What is Version Control? (with real-life analogy)
- Git vs GitHub — the key difference
- Core vocabulary: Repo, Commit, Branch, Clone, Fork, PR

⚙️ Module 2 — Setup & Installation

- Install Git on Windows, macOS, Linux
- Configure username and email
- Connect GitHub via HTTPS (Personal Access Token) and SSH keys

💻 Module 3 — Basic Commands

- `git init`, `git clone`, `git status`, `git add`, `git commit`
- `git log`, `git push`, `git pull`
- Complete daily workflow walkthrough

🌿 Module 4 — Branching & Merging

- Why branching matters
- `git branch`, `git switch`, `git merge`
- Merge conflict: what it is, how to fix it step by step

🤝 Module 5 — Collaboration

- Fork vs Clone
- Pull Requests + PR description template
- Code review best practices
- GitHub Issues for bug tracking

🌐 Module 6 — GitHub Features

- **GitHub Pages** — full 4-phase HTML hosting walkthrough
- **GitHub Actions** — CI/CD YAML pipeline setup
- **GitHub Copilot** — AI coding assistant (free for students)
- **Semantic Versioning** — v1.0.0, Major.Minor.Patch

🔧 Module 7 — Intermediate Topics

- `.gitignore` file (with security tips)
- Staging area deep dive
- Undoing changes: `restore`, `reset`, `revert`
- `git stash` — real-world use case

🚀 Module 8 — Advanced Topics

- Rebase vs Merge — when to use which
- Squashing commits — cleaning history before PR merge
- Cherry-pick — grab a single commit from another branch
- Git Flow, Feature Branch, Trunk-Based workflows

🏗️ Module 9 — 3 Projects

- 🟢 **Project 1 (Beginner):** Upload HTML project to GitHub
- 🟡 **Project 2 (Intermediate):** 2-person team collaboration + PR + conflict
- 🔴 **Project 3 (Advanced):** CI/CD auto-deployment with GitHub Actions

🎯 Module 10 — Interview Prep

- 8 conceptual Q&A (merge vs rebase, staging, conflicts, reset vs revert...)
- Viva quick-answer table
- 2 scenario-based questions (leaked API key, committed to wrong branch)

📋 Module 11 — Cheat Sheet

- 60+ commands quick reference
- Common errors & fixes table
- ⛔ What NOT to do in GitHub
- ⭐ Pro Tips from real developers

💼 Module 12 — Resume & Career

- GitHub profile optimization checklist
- How to list projects on your resume (with examples)
- What to say vs what NOT to say in interviews
- How to stand out with CI/CD knowledge

## ⭐ Pro Tips
```
💡  Commit often — small commits are easier to review and revert
💡  Always git pull before you git push
💡  Never commit .env files — API keys belong in environment variables
💡  Use meaningful commit messages: "fix: resolve login crash" not "fix"
💡  Pin your 6 best repos on your GitHub profile — recruiters look here
💡  Press '.' on any GitHub repo to open it in VS Code in the browser!
💡  git reflog is your safety net — run it if you think you lost work
```
